# 🎯 JavaScript Design Patterns Playground

> 🚀 **Полный справочник и реализация популярных паттернов проектирования** на языке **JavaScript**

---

✨ Этот репозиторий охватывает три ключевые категории паттернов:

- 🔨 **Порождающие (Creational)** — создание объектов разными способами
- 🧱 **Структурные (Structural)** — организация классов и объектов
- 🧩 **Поведенческие (Behavioral)** — управление взаимодействием между объектами

📚 Каждый паттерн снабжен:

- ✅ Понятными примерами
- 💬 Комментариями в коде
- ⚙️ Возможностью локального запуска
- 📁 Структурированной файловой системой

---

🎓 Подходит для:

- Начинающих и опытных разработчиков
- Подготовки к собеседованиям (Middle/Senior)
- Повышения насмотренности и архитектурного мышления

🔧 Стек: `JavaScript`, `Node.js`, (в планах — `TypeScript`, `React`, `Jest`, `Mermaid`)

## 📦 Установка и запуск

1. **Склонируй репозиторий**:

```bash
git clone https://github.com/your-username/designPatterns-js.git
cd design-patterns-js
```
2. **Запуск примеров**:

Ты можешь запускать каждый пример отдельно, например:

```bash
node creational/Factory/factory.js
node structural/Decorator/decorator.js
```

## 🧠 Что ты найдешь в этом проекте?

---

### 🔨 Порождающие паттерны:

- **Factory** – абстрагирует создание объектов
- **Builder** – поэтапное создание сложных объектов
- **Prototype** – клонирование объектов
- **Singleton** – один объект на всё приложение

---

### 🧱 Структурные паттерны:

- **Adapter** – переходник между несовместимыми интерфейсами
- **Decorator** – добавляет поведение "на лету"
- **Facade** – упрощает доступ к сложной системе
- **Proxy** – контроль доступа к объекту
- **Composite** – дерево из объектов
- **Bridge** – разделяет абстракцию и реализацию
- **Flyweight** – экономит память при повторяющихся объектах

---

### 🧩 Поведенческие паттерны:

- **Observer** – подписка/издатель
- **Strategy** – переключаемые алгоритмы
- **Command** – инкапсуляция команд
- **Iterator** – последовательный доступ к элементам
- **State** – объект меняет поведение в зависимости от состояния
- **Mediator** – посредник между объектами
- **Chain of Responsibility** – передача запроса по цепочке

---
